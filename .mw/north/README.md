## Accountability Records

### The Bootstrap Paradox

Veritas evaluates. But who evaluates veritas? This is the bootstrap paradox of an evaluation tool: the first evaluation must evaluate itself. The MMOT self-evaluation rispec (`rispecs/mmot-self-evaluation.rispec.md`) exists precisely for this — turning the tool's own methodology back on itself.

What we learned: self-evaluation is possible but requires external anchoring. The evaluation of veritas isn't "does veritas think it's good?" — it's "does veritas honestly name its own gaps?" The answer lives in the structural tensions we hold, not in the scores we generate.

### Wisdom Distilled

**On evaluation honesty**: The hardest part of building an evaluation tool is resisting the urge to make evaluations comfortable. Every design decision that softens output — "maybe consider" instead of "this is not acceptable" — undermines the tool's purpose.

**On structural tension**: Veritas holds tensions intentionally. The tension between "CLI tools planned but not built" was held honestly in KINSHIP.md until the tools were actually built. Premature resolution (claiming completion before it exists) is exactly what MMOT teaches us to catch.

**On integration**: A tool that evaluates in isolation evaluates nothing. Veritas's value emerges only in relationship — with coaia-narrative providing charts, with mia-code providing workflow integration, with medicine-wheel providing type coherence. The wisdom is relational, not autonomous.
